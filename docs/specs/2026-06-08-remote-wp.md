# Sentropic Remote — WP specs (2026-06-08)

Per-work-package spec: **objective · acceptance criteria · verification**.
Source of truth for what shipped in v0.5.0 and the follow-ups, since
`track` 0.2.2 cannot persist specs/criteria (the `item spec` / `accept
criterion` CLI subcommands are no-ops — exit 0, write nothing). Track holds the
items/realizations/decisions; this file holds their specs.

Verification legend: **T** = unit tests, **TC** = `typecheck --workspaces`,
**CI** = GHCR build green, **live** = exercised against the SCW cluster.

---

## WP1 — tmux scroll & accents (local + remote)
**Objective.** The wheel scrolls the *conversation*, not the CLI input history;
accents render correctly.
**Acceptance.**
- Wheel up enters copy-mode and scrolls the pane's 50k history (both Pod image
  `/etc/tmux.conf` and local server via `ensureScrollConfig`).
- No `Ctrl-b [` needed; `mouse on`, `focus-events on`.
- Accents render (exec-attach forces `LANG=C.UTF-8` + `tmux -u`).
**Verification.** live (local + both Pods, user-confirmed); image baked.

## WP2 — live-session registry + autoenrollment + layout snapshot
**Objective.** Stop guessing live sessions; record them and the launched layout.
**Acceptance.**
- `<configDir>/registry.json` with enroll/touch/markEnded/prune/listLive.
- `remote enroll --install-hooks` wires claude SessionStart/End hooks
  (idempotent, backs up settings.json); hook mode always exits 0.
- `remote run` auto-enrolls; `remote ls` shows `[registry]` vs `[guess]`.
- `remote restore` is registry-first (scan only fills gaps); writes
  `layout-last.json`; `remote layout` shows it.
**Verification.** T (registry, install-hooks idempotency, restore registry-first), live (hooks installed, smoke).

## WP3 — explicit GC of the shared RWX volume
**Objective.** Reclaim stale `ws-*` subdirs of the shared volume without ever
losing a conversation.
**Acceptance.**
- `remote workspace gc [--older-than N] [--apply --yes]`; dry-run by default.
- Janitor Pod mounts the volume ROOT (affinity to session nodes — the only
  placement guaranteed by the 1-FS-volume/node CSI).
- keep-list = every workspace the store knows, re-checked *inside* the Pod.
- `--apply` archives each candidate to on-volume `.trash/` (tar) BEFORE `rm`.
**Verification.** T (orchestrator janitor spec, CP route keep-derivation), TC, live (dry-run, janitor reaped).

## WP4 — fleet auth that heals itself
**Objective.** A control-plane restart loses nothing; credentials stay fresh
unattended.
**Acceptance.**
- Agent announce carries `home` + `startupArgs` + `cliSessionId`.
- On refresh, the CP rebuilds `--resume` from the *freshest* `cliSessionId`.
- `remote refresh --soft --all [--watch <min>]`; respawn gated on a sha256 of
  the credential files only (config churn like `.claude.json` doesn't respawn);
  a dead Pod CLI is revived even when creds are unchanged.
- `--watch` re-ensures the tunnel each pass (survives a CP redeploy).
**Verification.** T (announce parity, resume-rewrite, multi-session/gating/watch), TC, live (CP restart → re-announce; 2-pass unchanged; dead-pane revive).

## WP5 — single-writer guard per conversation
**Objective.** Never let two writers open the same conversation `.jsonl`.
**Acceptance.**
- `remote run -r <conv>` and `remote migrate forward [-r <conv>]` refuse when a
  live writer (local registry or remote `cliSessionId`) already holds it.
- Loud `--force` override; message names where it's open.
**Verification.** T (conv-guard pure + wiring run/migrate, bare `-r` resolution), TC.

## WP6 — `remote sync` + `remote diff`
**Objective.** Inspect and realign local↔Pod without losing lines.
**Acceptance.**
- `remote diff [id]`: conversation log (`--session`, default) or git workspace
  state (`--files`: HEAD/branch/modified names; content never transferred).
- `remote sync <id> --session push|pull`: base64 once, line-count guard,
  `.bak-<epoch>` backup, `--force` to override the ahead-guard.
**Verification.** T (decideSyncAction matrix, gitAlignment, encoding), TC.

## WP7 — h2a wiring
**Objective.** Deported agents reachable/wakeable through the h2a file bus.
**Acceptance.**
- `remote run --h2a` (config `h2a.{enabled,command}`) starts `h2a mcp-serve
  --auto-open --auto-upgrade --wake local-tmux` in a side tmux window; profile
  names the main window; binary-absent → warning, run never fails.
- `remote h2a bridge [id] [--watch]` transports envelopes Pod↔local (pull
  emitted, push addressed, idempotent by filename, never deletes; scaffolds the
  Pod `.h2a` if missing).
**Verification.** T (tmux window args, config round-trip, bridge sync pure), TC, live (envelope local→Pod round-trip).

## WP8 — image & infra
**Objective.** Kill cold-pull pain; CP store durable.
**Acceptance.**
- `runtime-slim` image variant (1.26GB vs 2.47GB fat — no Go/Rust/build tools)
  published `:vX-slim`/`:main-slim`; fat stays the default (last stage).
- Pre-pull DaemonSet (`deploy/scw/40-prepull.yaml`, `make scw-prepull`) keeps
  the fat image warm on the session pool.
- CP store repopulates from agent re-announce after a restart.
**Verification.** docker build both stages (sizes measured), CI (slim tags), TC/T, live (CP restart durability).

## WP9 — `remote plugin`
**Objective.** One npm plugin (CLI + MCP) installed everywhere, local and Pods.
**Acceptance.**
- `remote plugin add <pkg>` / `ls` / `sync`; registers MCP server(s) with
  claude + codex + agy (`~/.gemini/config/mcp_config.json`); MCPs as
  `node <realpath>` (npm-global symlink breaks bare-bin entrypoints).
- `sync` installs + registers into every live Pod, idempotent.
**Verification.** T (parse/merge/upsert/pod-script/injection guards), TC, live (`@sentropic/track` registered local + 2 Pods).
